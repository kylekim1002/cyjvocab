# 프로젝트 진화 매트릭스 (유지 vs 새 설계)

실사용자 배포 전, **기능·도메인은 유지**하고 **학생 반응속도·신뢰성**을 위해 다시 짤 때 참고용 표입니다.

---

## 범례

| 표기 | 의미 |
|------|------|
| **유지** | 비즈니스 요구·데이터 모델·화면 목적은 그대로 |
| **재설계** | 동작은 같게 보이게 하되, 상태·API·인프라·저장 방식은 새로 정의 |
| **검토** | 유지할지 통합할지 결정 필요 |

---

## 1. 도메인·데이터 (Prisma / Supabase)

| 항목 | 유지 | 재설계·검토 |
|------|------|-------------|
| 역할 (SUPER_ADMIN / MANAGER / STUDENT) | ✅ | — |
| 캠퍼스·클래스·학생·코드값 | ✅ | — |
| 학습 모듈·문항 (`LearningModule` / `LearningItem`, `payloadJson`) | ✅ 구조 | 재설계: payload 필드 규약 문서화, 큰 JSON vs 이벤트 로그 검토 |
| 배정 (`ClassAssignment` / `ClassAssignmentModule`, source) | ✅ | — |
| 진행·세션 (`StudentAssignmentProgress`, `StudySession`) | ✅ 존재 | 재설계: 스냅샷 저장 주기, 멱등·revision, 이벤트 테이블 여부 |
| 음원 풀 (`WordAudio`) | ✅ | 재설계: 업로드·매칭 키 규칙만 문서로 고정 |
| 점수·오답·통계·세션 완료 | ✅ 비즈니스 | 재설계: 집계 시점(실시간 vs 배치) |
| Storage 버킷 `learning-audio` | ✅ | 검토: 캐시 헤더·경로 규칙 |

## 2. 학생 화면·학습 UX

| 항목 | 유지 | 재설계·검토 |
|------|------|-------------|
| 단계: 단어목록 / 단어학습 / 암기 / 쓰기 / 테스트 / 복습 | ✅ | 재설계: 단일 state machine·reducer로 표현 |
| 다음·이전·보기 선택·듣기·완료 | ✅ 사용자 경험 | 재설계: 클릭 = 로컬만 즉시; 서버 = 디바운스·완료 시 |
| 학습 진입 URL (`/student/learn/...`) | ✅ 목적 | 검토: 학습만 별도 SPA/번들 분리 여부 |
| 긴 제목·단계 라벨 헤더 레이아웃 | ✅ (이미 개선됨) | 유지 |
